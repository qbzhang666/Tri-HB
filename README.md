# Tri-HB

Integrated Streamlit workspace for Triaxial Hopkinson Bar test design,
experimental data analysis, stress-wave interpretation, and DEM-oriented
damage validation.

## Quick start

```powershell
py -3.13 -m pip install -r requirements.txt
py -3.13 -m streamlit run tri_hb_integrated.py
```

Then open the URL Streamlit prints in your browser. See
[`docs/local_setup.md`](docs/local_setup.md) for the long-form setup notes,
PowerShell snippets, and alternate-port launch commands.

## What the app does

The integrated app navigates four steps under one shared experimental setup:

1. **Setup, simulator and data** — material, specimen, bar, and loading
   configuration; run the four-mode dynamic simulator; **or** reduce uploaded
   bar-gauge / direct stress-strain data with the same setup.
2. **Wave model** — pulse timing, travel time, equilibrium window,
   wave-superposition factor, and a wave-vs.-damage regime classifier.
3. **Stress path and analysis** — *p*-*q*-*θ* paths, invariant histories,
   and comparison against Step 1 reduced data.
4. **Damage model and DEM validation** — Perzyna-type damage evolution,
   stiffness and wave-speed degradation, energy balance, DEM/experimental
   descriptors, and CSV export.

Loading modes covered by the Step 1 simulator: gas-gun uniaxial, EM uniaxial,
EM asynchronous triaxial (with full per-axis three-wave analysis on each of
the three driven axes), and EM symmetric multidirectional (±X, ±X±Y, full
hydrostatic ±X±Y±Z).

## Active runtime dependency map

| File at repo root | Loaded by |
|---|---|
| `tri_hb_integrated.py` | the Streamlit entry point |
| `Tri-HB.py` | Step 1 → *Test design and simulator* |
| `wave_damage.py` | Steps 2 → *Wave model*, 3 → *Stress path*, 4 → *Damage model* (one module, three different tab orderings) |

Nothing else in the repo is touched while the app is running. `scripts/`
is used only when you regenerate Handbook figures; `archive/` is read-only
reference material.

## Python dependencies

See `requirements.txt`. The active runtime files actually import:
`streamlit`, `numpy`, `pandas`, `plotly`, `matplotlib`, and the `openpyxl`
engine (for XLSX export). Older entries in the file (`scipy`,
`scikit-learn`, `seaborn`, `mplstereonet`, `XlsxWriter`) are not currently
imported by any active file and can be pruned if you want a leaner install.
